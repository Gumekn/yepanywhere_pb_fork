# 重启功能说明

## 当前问题

### ✅ 问题 1：浏览器重启按钮智能重构建（已修复）

**新功能：**
- 点击浏览器中的"重启服务器"按钮
- **自动执行：构建最新代码 → 重启服务端**
- 开发模式会切换到生产模式
- 生产模式会重新构建后重启

**工作流程：**

#### 开发模式 → 构建并启动生产模式

1. 检测到当前运行开发模式（`build.source === "dev"`）
2. 调用部署系统的 `"server"` 动作
3. 自动执行：
   - 停止开发模式进程
   - `pnpm build`（构建生产代码到 `dist/npm-package/`）
   - 启动生产模式服务端
4. 显示提示："Stopping dev mode, building and starting production mode..."

#### 生产模式 → 重新构建并重启

1. 检测到当前运行生产模式（`build.source === "bundle"`）
2. 调用部署系统的 `"server"` 动作
3. 自动执行：
   - 停止当前服务端
   - `pnpm build`（重新构建最新代码）
   - 启动生产模式服务端
4. 显示提示："Rebuilding and restarting server..."

**注意事项：**
- 每次重启都会重新构建，确保运行的是最新代码
- 构建 + 重启需要 1-2 分钟（包含构建时间）
- 重启后需要手动刷新浏览器以加载新的前端代码

### 问题 2：yep.sh 重构建功能不完整

**现象：**
- 运行 `./yep.sh` 选择"7) 重构建项目"
- 构建成功，但浏览器中版本没有变化

**原因：**
重构建只执行了：
```bash
pnpm lint      # 代码检查
pnpm typecheck # 类型检查
pnpm build     # 构建到 dist/npm-package/
```

但**没有重启服务端进程**，所以：
- ✅ 已构建包已更新（磁盘上的文件）
- ❌ 服务端还在运行旧代码（进程未重启）
- ❌ 客户端还显示旧版本（浏览器未刷新）

**已修复：**
现在重构建后会询问：
```
是否重启服务端以应用新代码？
1) 重启生产模式
2) 重启开发模式
3) 稍后手动重启
```

## 推荐工作流程

### 开发时（日常修改代码）

```bash
# 1. 启动开发模式
./yep.sh
选择 1) 启动开发模式
选择 1) 前台运行

# 2. 修改代码
# 前端代码改动会自动热更新（Vite HMR）
# 后端代码改动需要手动重启：Ctrl+C 然后重新运行 pnpm dev

# 3. 测试完成后 Ctrl+C 停止
```

### 测试生产构建

```bash
# 1. 重构建项目
./yep.sh
选择 7) 重构建项目

# 2. 重构建成功后选择重启
选择 2) 重启开发模式  # 如果想继续开发
# 或
选择 1) 重启生产模式  # 如果想测试生产环境

# 3. 刷新浏览器
Cmd+R 或 Ctrl+R
```

### 部署到生产

```bash
# 1. 确保代码已提交
git add .
git commit -m "更新说明"

# 2. 重构建
./yep.sh
选择 7) 重构建项目

# 3. 重启生产模式
选择 1) 重启生产模式

# 4. 验证版本
访问 http://localhost:3400/settings/about
确认版本号已更新
```

## 版本概念复习

| 版本显示 | 含义 | 位置 | 如何更新 |
|---------|------|------|----------|
| **仓库版本** | 源代码版本 | `package.json` | `git pull` |
| **已构建包** | 编译产物 | `dist/npm-package/` | `pnpm build` |
| **服务端** | 运行中的后端 | Node.js 进程内存 | 重启服务端 |
| **客户端** | 运行中的前端 | 浏览器内存 | 刷新浏览器 |

**更新流程：**
```
修改源代码 → pnpm build → 重启服务端 → 刷新浏览器
(仓库版本)    (已构建包)    (服务端)      (客户端)
```

## 常见问题

### Q: 为什么重构建后版本还是旧的？
A: 需要重启服务端并刷新浏览器。现在 `yep.sh` 重构建后会自动询问是否重启。

### Q: 开发模式下如何重启服务端？
A: 在终端中按 `Ctrl+C` 停止，然后重新运行 `pnpm dev` 或 `./yep.sh`。

### Q: 浏览器中的"重启服务器"按钮何时能用？
A: 生产模式下可用。开发模式下不推荐使用（会导致进程停止但不自动重启）。

### Q: 如何知道当前是开发模式还是生产模式？
A: 
- 访问 `/api/version`，查看 `build.source` 字段
- `"dev"` = 开发模式
- `"bundle"` = 生产模式

### Q: 重启服务器和重构建的区别？
A:
- **重构建**：`lint → typecheck → build`，生成新的编译产物
- **重启服务器**：停止旧进程，启动新进程，加载最新的编译产物

通常需要先重构建，再重启。
